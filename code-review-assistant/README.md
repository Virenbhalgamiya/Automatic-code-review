# Automated Code Review Assistant

A production-quality static analysis API and database persistence service that reviews Python source files using Python's built-in Abstract Syntax Tree (`ast`) module and persists the quality metrics in PostgreSQL as native JSONB. It acts as an automated quality-gate service suitable for integration into CI/CD pipelines, checking files for common bad practices, undocumented blocks, and hardcoded credentials.

## Getting Started

### Prerequisites
- Docker & Docker Compose installed.

### How to Run Locally
Start the application and PostgreSQL database using the one-command startup script or docker-compose directly:

```bash
# On Linux/macOS
./run.sh

# On Windows (PowerShell)
.\run.ps1

# Or run manually
docker-compose up --build
```
The FastAPI server will boot and be accessible at `http://localhost:8000`. You can visit `http://localhost:8000/docs` to view the interactive API swagger documentation.

### How to Run Tests
To run all tests (isolated AST engine unit tests and full database integration tests) inside the Docker container, run the following command:

```bash
docker-compose run api pytest -s
```

## Static Analysis Rules Checked

1. **Unused Imports**: Checks for imported modules or names that are never loaded or referenced.
2. **Long Functions**: Flags any function or method whose body block exceeds 20 lines.
3. **Missing Docstrings**: Detects functions, methods, or classes that lack docstring declarations.
4. **Hardcoded Secrets**: Identifies string literals assigned to variables whose names contain `password`, `secret`, `key`, `token`, or `api` (case-insensitive).
5. **Bare Excepts**: Warns against generic `except:` statements catching all events instead of specific exceptions.
6. **Missing Type Hints**: Triggers when arguments in function definitions lack type annotations (skips `self` and `cls` parameters).
